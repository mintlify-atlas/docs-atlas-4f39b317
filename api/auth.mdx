---
title: "Auth API — Register, Login, and Manage Users"
sidebarTitle: "Auth"
description: "REST endpoints for user registration, authentication, and admin user management in Yakult App. Returns JWT tokens valid for 30 days."
---

The Auth API handles everything from creating new accounts to managing existing users. On registration, roles are assigned automatically based on the email domain — addresses ending in `@upa.edu.mx` become **Master** accounts, while all other addresses receive the **Promotor** role. Every successful registration or login returns a signed JWT valid for 30 days that must be included as a Bearer token on protected routes.

---

### POST /api/auth/registro

Registers a new user account and returns the created user object along with a JWT.

<ParamField body="nombre" type="string" required>
  Full name of the user.
</ParamField>

<ParamField body="correo" type="string" required>
  Email address. Accounts registered with an `@upa.edu.mx` address are automatically assigned the **Master** role; all other addresses receive the **Promotor** role.
</ParamField>

<ParamField body="contrasena" type="string" required>
  Plain-text password. Stored as a bcrypt hash.
</ParamField>

**Example request**

```bash
curl -X POST http://localhost:3000/api/auth/registro \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Ana García",
    "correo": "ana.garcia@upa.edu.mx",
    "contrasena": "secreto123"
  }'
```

**Example response**

```json
{
  "usuario": {
    "id": 1,
    "nombre": "Ana García",
    "correo": "ana.garcia@upa.edu.mx",
    "rol": "Master"
  },
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

<ResponseField name="usuario" type="object">
  The newly created user record.

  <Expandable title="usuario fields">
    <ResponseField name="id" type="number">Unique numeric ID of the user.</ResponseField>
    <ResponseField name="nombre" type="string">Display name.</ResponseField>
    <ResponseField name="correo" type="string">Email address.</ResponseField>
    <ResponseField name="rol" type="string">Assigned role: `Master`, `Promotor`, or `Repartidor`.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="token" type="string">
  Signed JWT. Include as `Authorization: Bearer <token>` on protected requests. Expires after 30 days.
</ResponseField>

**Errors**

| Status | Reason |
|--------|--------|
| `400` | One or more required fields are missing. |
| `400` | A user with that email address already exists. |

### POST /api/auth/login

Authenticates an existing user and returns a fresh JWT.

<ParamField body="correo" type="string" required>
  Registered email address.
</ParamField>

<ParamField body="contrasena" type="string" required>
  Account password.
</ParamField>

**Example request**

```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "correo": "ana.garcia@upa.edu.mx",
    "contrasena": "secreto123"
  }'
```

**Example response**

```json
{
  "usuario": {
    "id": 1,
    "nombre": "Ana García",
    "correo": "ana.garcia@upa.edu.mx",
    "rol": "Master"
  },
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

<ResponseField name="usuario" type="object">
  The authenticated user record.

  <Expandable title="usuario fields">
    <ResponseField name="id" type="number">Unique numeric ID.</ResponseField>
    <ResponseField name="nombre" type="string">Display name.</ResponseField>
    <ResponseField name="correo" type="string">Email address.</ResponseField>
    <ResponseField name="rol" type="string">Role: `Master`, `Promotor`, or `Repartidor`.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="token" type="string">
  Signed JWT valid for 30 days.
</ResponseField>

**Errors**

| Status | Reason |
|--------|--------|
| `400` | `correo` or `contrasena` field is missing. |
| `401` | Email not found or password is incorrect. |
| `403` | The account has been disabled by an administrator. |

### GET /api/auth/usuarios

Returns a list of all registered users. Intended for **Master** users to monitor and manage accounts.

**Example request**

```bash
curl http://localhost:3000/api/auth/usuarios
```

**Example response**

```json
[
  {
    "id": 1,
    "nombre": "Ana García",
    "correo": "ana.garcia@upa.edu.mx",
    "rol": "Master",
    "activo": true,
    "creado_en": "2025-01-10T08:00:00.000Z"
  },
  {
    "id": 2,
    "nombre": "Carlos Ruiz",
    "correo": "carlos.ruiz@gmail.com",
    "rol": "Promotor",
    "activo": true,
    "creado_en": "2025-01-11T09:15:00.000Z"
  }
]
```

<ResponseField name="id" type="number">Unique numeric ID of the user.</ResponseField>
<ResponseField name="nombre" type="string">Display name.</ResponseField>
<ResponseField name="correo" type="string">Email address.</ResponseField>
<ResponseField name="rol" type="string">Role: `Master`, `Promotor`, or `Repartidor`.</ResponseField>
<ResponseField name="activo" type="boolean">Whether the account is currently active and can log in.</ResponseField>
<ResponseField name="creado_en" type="string">ISO 8601 timestamp of when the account was created.</ResponseField>

### PUT /api/auth/usuarios/:id

Enables or disables a user account. Disabled users receive a `403` on login.

<ParamField path="id" type="number" required>
  Numeric ID of the user to update.
</ParamField>

<ParamField body="activo" type="boolean" required>
  Pass `true` to re-enable the account or `false` to disable it.
</ParamField>

**Example request**

```bash
curl -X PUT http://localhost:3000/api/auth/usuarios/2 \
  -H "Content-Type: application/json" \
  -d '{ "activo": false }'
```

**Example response**

```json
{ "ok": true }
```

**Errors**

| Status | Reason |
|--------|--------|
| `404` | No user found with the given ID. |

---

### PUT /api/auth/usuarios/:id/rol

Changes the role of an existing user.

<ParamField path="id" type="number" required>
  Numeric ID of the user whose role should be updated.
</ParamField>

<ParamField body="rol" type="string" required>
  New role to assign. Must be one of `Master`, `Promotor`, or `Repartidor`.
</ParamField>

**Example request**

```bash
curl -X PUT http://localhost:3000/api/auth/usuarios/2/rol \
  -H "Content-Type: application/json" \
  -d '{ "rol": "Repartidor" }'
```

**Example response**

```json
{ "ok": true }
```

**Errors**

| Status | Reason |
|--------|--------|
| `400` | The value of `rol` is not one of the three valid options. |
| `404` | No user found with the given ID. |

---

### DELETE /api/auth/usuarios/:id

Permanently deletes a user account. This action cannot be undone.

<ParamField path="id" type="number" required>
  Numeric ID of the user to delete.
</ParamField>

**Example request**

```bash
curl -X DELETE http://localhost:3000/api/auth/usuarios/2
```

**Example response**

```json
{ "ok": true }
```

**Errors**

| Status | Reason |
|--------|--------|
| `404` | No user found with the given ID. |
