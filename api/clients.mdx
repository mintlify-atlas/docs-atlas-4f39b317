sidebarTitle: "Clients"
description: "REST endpoints to manage client records including name, phone number, address, and active status. Active clients can receive WhatsApp order updates."
---

The Clients API manages the customer directory for Yakult distributors. Each client record stores the essential contact details needed for order delivery — name, phone number, and address. The `activo` flag determines whether a client appears in order-creation flows; toggling it off lets you retain historical data without exposing the client to new orders. No authentication is required on any clients endpoint.

---

### GET /api/clientes

Returns all client records ordered alphabetically by name, including both active and inactive clients.

**Example request**

```bash
curl http://localhost:3000/api/clientes
```

**Example response**

```json
[
  {
    "id": 1,
    "nombre": "Minisuper Torres",
    "telefono": "6649876543",
    "direccion": "Av. Revolución 120, Tijuana, BC",
    "activo": 0,
    "creado_en": "2025-01-06T11:30:00.000Z"
  },
  {
    "id": 2,
    "nombre": "Tienda López",
    "telefono": "6641234567",
    "direccion": "Calle Morelos 45, Tijuana, BC",
    "activo": 1,
    "creado_en": "2025-01-05T10:00:00.000Z"
  }
]
```

<ResponseField name="id" type="number">Unique numeric ID of the client.</ResponseField>
<ResponseField name="nombre" type="string">Business or customer name.</ResponseField>
<ResponseField name="telefono" type="string">Phone number used for WhatsApp order updates.</ResponseField>
<ResponseField name="direccion" type="string">Delivery address.</ResponseField>
<ResponseField name="activo" type="number">Account status as a TINYINT: `1` = active, `0` = inactive.</ResponseField>
<ResponseField name="creado_en" type="string">ISO 8601 timestamp of when the record was created.</ResponseField>

### POST /api/clientes

Creates a new client record. Newly created clients are active by default. No authentication is required.

<ParamField body="nombre" type="string" required>
  Business or customer name, e.g. `"Tienda López"`.
</ParamField>

<ParamField body="telefono" type="string" required>
  Phone number. Used to send WhatsApp notifications on order status changes.
</ParamField>

<ParamField body="direccion" type="string" required>
  Full delivery address.
</ParamField>

**Example request**

```bash
curl -X POST http://localhost:3000/api/clientes \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Tienda López",
    "telefono": "6641234567",
    "direccion": "Calle Morelos 45, Tijuana, BC"
  }'
```

**Example response**

```json
{
  "id": 1,
  "nombre": "Tienda López",
  "telefono": "6641234567",
  "direccion": "Calle Morelos 45, Tijuana, BC",
  "activo": true
}
```

<ResponseField name="id" type="number">Auto-generated ID assigned to the new client.</ResponseField>
<ResponseField name="nombre" type="string">Client name as stored.</ResponseField>
<ResponseField name="telefono" type="string">Phone number as stored.</ResponseField>
<ResponseField name="direccion" type="string">Delivery address as stored.</ResponseField>
<ResponseField name="activo" type="boolean">Always `true` for a newly created client.</ResponseField>

### PUT /api/clientes/:id

Updates the name, phone number, and address of an existing client. Supply all three fields. No authentication is required.

<ParamField path="id" type="number" required>
  Numeric ID of the client to update.
</ParamField>

<ParamField body="nombre" type="string" required>
  Updated client name.
</ParamField>

<ParamField body="telefono" type="string" required>
  Updated phone number.
</ParamField>

<ParamField body="direccion" type="string" required>
  Updated delivery address.
</ParamField>

**Example request**

```bash
curl -X PUT http://localhost:3000/api/clientes/1 \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Tienda López Renovada",
    "telefono": "6641234567",
    "direccion": "Calle Morelos 45, Local 2, Tijuana, BC"
  }'
```

**Example response**

```json
{ "ok": true }
```

---

### PUT /api/clientes/:id/activo

Enables or disables a client without deleting their record. Inactive clients are excluded from new order creation. No authentication is required.

<ParamField path="id" type="number" required>
  Numeric ID of the client to update.
</ParamField>

<ParamField body="activo" type="boolean" required>
  Pass `true` to re-activate the client or `false` to deactivate them.
</ParamField>

**Example request**

```bash
curl -X PUT http://localhost:3000/api/clientes/1/activo \
  -H "Content-Type: application/json" \
  -d '{ "activo": false }'
```

**Example response**

```json
{ "ok": true }
```

---

### DELETE /api/clientes/:id

Permanently deletes a client record. Consider using `PUT /api/clientes/:id/activo` with `activo: false` to preserve order history. No authentication is required.

<ParamField path="id" type="number" required>
  Numeric ID of the client to delete.
</ParamField>

**Example request**

```bash
curl -X DELETE http://localhost:3000/api/clientes/1
```

**Example response**

```json
{ "ok": true }
```
